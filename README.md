## 关于本项目
这是一个基于wails的更新器与内容管理工具，主要为服务器设计，前端使用react+typescript，逻辑部分为go，在对应系统上直接使用webview减少对客户端系统的资源占用.<br/>

## 如果您要开发/自己构建
- 先下载对应环境(go1.23, node25),这里不再赘述<br/>
- 再安装wails，`go install github.com/wailsapp/wails/v2/cmd/wails@latest`<br/>
- 运行`wails doctor`检查系统环境，确定系统并无缺失依赖。
- 在项目根目录运行`wails dev`即可看到弹出窗口。如果需要，在`http://localhost:34115/` 可打开项目前端网页<br/>
- 运行`wails build`即可生成可执行文件

## 对于使用者/整合包开发者
一般来说，文件json和配置json都应保存于`/config/`目录下，运行时将自动读取并加载。但只有配置json会自动生成，而文件json需要您手动编写。<br/>
文件json请放置到`/config/files.json`，配置json请放置到`/config/config.json`，若您需要在整合包中使用，请将配置一并打包<br/>
若需要使用更新功能和延迟检测功能，需要您自己在http服务器中托管文件，并将相应的url填入配置文件中分发。<br/>
所有的示例文件均在`/example/`目录下，请自行参考查看。
